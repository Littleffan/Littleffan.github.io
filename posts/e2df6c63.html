遇到的小问题：

1.win11中鼠标右键Git Bash Here选项消失。

可以参考这篇大神的文章鼠标右键没有git bash here，右键添加git bash here并增加图标

2.将valine替换为Gitalk

valine评论系统失效，[valine](jsDelivr 的 valine CDN - 用于 npm 和 GitHub 的 CDN)，直接没法显示。看了很多网上分享的文章，也改了我能懂的所有方式，仍然无法让它显示出来，所以直接放弃了。选择了Gitalk，最开始也是谷歌搜了之后直接参考，但是后来才发现总是报错误：typeerror: cannot read properties of undefined (reading ‘gitalk’)。解决不掉，看到hexo博客 NexT 7.7.0以后版本 gitalk配置才解决了，网上大部分教程都是老版本的，照着做反而会有问题。NexT（v7.7.0）新版本已经集成Gitalk所有只有简单的三步就可以搞定。

  • 注册OAuth application（访问https://github.com/settings/applications/new ）

    相关信息按以下方法填写：

  • 修改NexT的config配置文件

  • 然后hexo clean，hexo g， hexo d三连后再登录关联账号即可（备注hexo d完了之后还要管理员手动初始化一下）

3.安装abbrlink的插件，生成永久短链接

部署后新建的文章点开均指向同一个链接。网络上搜索后发现是没安装abbrlink的插件，于是“npm install hexo-abbrlink –save”。再次hexo g发现一直报错，点开文章，确实是新建的文章都生成了abbrlink链接。但是莫名gitalk出现undefined报错，bluebird也报错。网络上搜不到相关信息，尝试之后发现只要我执行“npm install hexo-abbrlink –save”命令，node_modules文件夹中同时会创建一个hexo-plugin-gitalk的文件夹，只要把这个hexo-plugin-gitalk文件夹删掉，再次重新生成部署就没问题了。

参考github：hexo-abbrlink

4.添加外部链接

添加外部链接的时候如果是 [百度一下] (www.baidu.com)就会出问题，无法加载例如[**百度一下**](www.baidu.com)。需要更改为[**百度一下**] (https://www.baidu.com)，例如[**百度一下**](https://www.baidu.com)。
